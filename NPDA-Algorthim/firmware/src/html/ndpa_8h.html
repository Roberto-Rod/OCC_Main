◆ ASSUME_VARIABLE_SIZE_BLOCKS

#define ASSUME_VARIABLE_SIZE_BLOCKS   1

Assuming that blocks can contain different number of scans. Setting this to 1, will calculate the max value and position in the averaged array for every single call to ndpa_scan, while setting it to 0 will expect all blocks to contain MAX_SCANS scans and avg max only calculated after the last scan is added to the block.

Note
Set to 1 if using restricted data sets and the calling module is NOT filtering out scans with 0 offset values.

◆ MAX_POSITIONS

#define MAX_POSITIONS   5000

Number of measurements within a scan.

Warning

◆ MAX_WINDOW_SIZE

#define MAX_WINDOW_SIZE   1000

◆ USE_DOUBLE_PRECISION

#define USE_DOUBLE_PRECISION   0

◆ USE_INTERPOLATED_POS_BUFFER

#define USE_INTERPOLATED_POS_BUFFER   0

Use __ndpa_x in calculations.

Note
1 to use __ndpa_x
0 to use __ndpa_x_start and INTERPOLATION_DISTANCE instead of __ndpa_x
Although 0 value saves RAM space, but takes about 5% longer and result is very slightly different (e.g. 535.778723um vs 535.778700)!
Does not work with restricted scans!
